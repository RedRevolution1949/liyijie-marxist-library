#### 如何在本地部署本知识库？

为了避免被强迫下架的风险，我希望读者在空间足够的情况下尽可能部署在本地。

如果想要这样，建议你安装 `Git` 客户端，然后在磁盘空间足够的地方运行（在 `cmd` 或 `bash` 中）：

```
git clone --depth=1 https://github.com/RedRevolution1949/liyijie-marxist-library.git
cd liyijie-library
git submodule update --init --recursive
```

完成之后，仓库即被本地部署到 `liyijie-library` 文件夹。

进行了本地部署的读者，如果发现本仓库“消失”了，请协助补档，非常感谢。

#### 如何上传大文件？

以 Github 平台为例，通过浏览器上传的文件大小限制为 25MB，通过 `git` 上传的文件大小限制为 100MB。

如果你需要上传更大的文件，请使用  `git lfs` 功能。

Github 的仓库大小（`.git` 文件夹）建议在 1~5GB 以下，由于 `git` 会保留每次推送的差异，建议你不要反复删除、添加新文件。

你可以强制覆盖之前的提交来重置这个大小为仓库当前版本的实际大小，但是要注意先前的子模块绑定会失效，需要重新更新新版本。

对于很多文件，建议你分仓库存储，并以子模块形式绑定。 

为了避免上传问题，所有文档必须经过压缩，详见附录 B。

#### 如何向仓库添加文件？

提示：如果你实在不会，请提出 Issue，我可以协助你完成以下步骤。

###### 第一步：创建子仓库

由于 Github 的空间限制，我们将一些占用空间较大的仓库作为子模块（`submodule`）链接在主库里。

所以请创建一个新的仓库，上传文件。

我们支持的文件类型为 `.pdf`, `.htm/.html`, `.txt`, `.md`。

###### 第二步：提交 Pull Request

在你自己的 `fork` 仓库中，以子模块形式将你的文件仓库添加到 [files/](files/) 目录下。

```
# 你现在在主仓库根目录
git submodule add <URL> files/仓库名称
```

然后在 `Git` 托管网站中，提交 `Pull Request`，请求合并仓库。

经过审核之后，你的子仓库即被提交到了主仓库。

想要更新你的子仓库，详阅附录 A。

上传的文件必须经过压缩，详阅附录 B。

如果文件过大，审核有可能不会通过，并要求你进行压缩。

###### 附录 A：更新子仓库

在你的子仓库文件更新的情况下，主仓库并不会跟随更新，而是会保留原先的版本不变。

在确保重新进行文字识别和索引工作后，请更新子模块对应的版本，再次提交   `Pull Request`。

##### 附录 B：压缩

为了尽量减小 PDF 文件的尺寸，建议你将 `files/compress.sh` 复制到自己的文件夹下运行，

压缩之后，删除该脚本并提交。

Windows 平台下，请复制 `files/compress.bat`。

需要的软件：`ocrmypdf`, `tesseract`, `jbig2enc`, `pngquant`。

过大的文件会给上传和储存添麻烦。

如果压缩脚本没能完成压缩，可以使用其他压缩器进行压缩（本地压缩软件或在线压缩平台）。

我个人使用的是 `PDF Compressor 2.7.0.0`, `iLovePDF` 和 `OCRMyPDF`，效果很好。

对于极少数普通方法无法压缩的文件，可以将他们二次打印为 PDF 文件（以黑白颜色打印）。

然后将打印出的文件使用 OCRMyPDF 脚本二次压缩。

压缩的底线是字能看清，图片再怎么样也无所谓。
